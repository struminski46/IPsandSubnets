01

O que é uma Sub-rede (Subnet)?

Uma rede IP classful (Classe A, B ou C) é um bloco enorme de endereços consecutivos. Como um sanduíche de 2 km — muito grande para usar inteiro. A solução é fatiar essa rede em pedaços menores chamados sub-redes.

🔒

Regra 1 — Endereços no mesmo link (LAN, VLAN, WAN ponto-a-ponto) ficam na mesma sub-rede.

🔀

Regra 2 — Endereços em links diferentes ficam em sub-redes diferentes, separados por pelo menos um roteador.

Onde é necessária uma sub-rede?

  • Cada VLAN → 1 sub-rede
  • Cada link serial ponto-a-ponto → 1 sub-rede
  • Cada link Ethernet WAN (EoMPLS) → 1 sub-rede
💡 Exemplo: rede com 1 site central (12 VLANs) + 3 filiais (2 VLANs cada) + 3 links WAN = 12 + 6 + 3 = 21 sub-redes.

Estrutura do Endereço IPv4

Um endereço IPv4 tem 32 bits. Dependendo do contexto, pode ser visto com 2 ou 3 partes:

Visão Classless (2 partes)

PREFIXO (P bits)
HOST (H bits)

Visão Classful (3 partes) — N + S + H = 32

REDE (N)
SUBNET (S)
HOST (H)
Rede — definida pela Classe (A=8, B=16, C=24 bits)
Subnet — bits "tomados emprestado" dos bits de host
Host — identifica cada dispositivo na sub-rede

Exemplo — Classe A, /24

N=8
S=16
H=8

Ex: 10.1.1.0/24

Exemplo — Classe B, /24

N=16
S=8
H=8

Ex: 172.16.1.0/24

Três formas de representar a mesma máscara:

05

Endereços IP Privados (RFC 1918)

A Internet ficou sem endereços IPv4 públicos. A solução: NAT + IPs Privados. Empresas podem reutilizar os mesmos endereços internamente porque o NAT traduz para IPs públicos na saída para a Internet.

Classe Redes Privadas # Redes
A 10.0.0.0 1
B 172.16.0.0 → 172.31.0.0 16
C 192.168.0.0 → 192.168.255.0 256
💡 Cerca de 50% das empresas usam a rede privada Classe A 10.0.0.0 — mesmo sendo gigante, não há penalidade por escolhê-la para uma rede pequena.
06

Cálculos Fundamentais

Hosts utilizáveis por sub-rede
2H − 2

H = número de bits de host. Subtrai 2 porque o menor número = Subnet ID e o maior número = Broadcast — ambos reservados.

Número de sub-redes (1 máscara)
2S

S = bits de subnet (bits emprestados dos bits de host). Só válido quando uma única máscara é usada em toda a rede classful.

Exemplo: Rede 172.16.0.0, máscara /24

N = 16
S = 8
H = 8
Hosts/subnet: 2⁸ − 2 = 254
Sub-redes: 2⁸ = 256

Escolhendo a máscara — Exemplo Prático

Precisa: 200 sub-redes e 200 hosts/sub-rede na rede Classe B 172.16.0.0

Bits de subnet (S): 2⁷=128 (insuficiente) → 2⁸=256 ✓ → S=8

Bits de host (H): 2⁷−2=126 (insuficiente) → 2⁸−2=254 ✓ → H=8

Verificação: N+S+H = 16+8+8 = 32 bits ✓

Máscara resultante: 16 uns + 8 uns + 8 zeros = 255.255.255.0 (/24)

07

Encontrar Subnet ID e Broadcast

Conceito: o que define uma sub-rede

  • Subnet ID (menor número): identifica a sub-rede, não pode ser atribuído a hosts
  • Broadcast (maior número): enviado a todos os hosts da sub-rede, também reservado
  • Range utilizável: Subnet ID + 1 até Broadcast − 1

Método Decimal — Máscara Fácil (só 255 e 0)

Para máscaras como 255.0.0.0, 255.255.0.0, 255.255.255.0:

Subnet ID

255

Copie o octeto do endereço IP

0

Escreva 0

Broadcast

255

Copie o octeto do endereço IP

0

Escreva 255

IP:10.77.55.3Máscara:255.255.255.0
Subnet ID:10.77.55.0
Broadcast:10.77.55.255
Range:10.77.55.1 → 10.77.55.254

Método Decimal — Máscara Difícil (octeto de interesse)

Quando há um octeto que não é 0 nem 255, esse é o octeto de interesse:

Se máscara = 255: copie o octeto do IP

Se máscara = 0: escreva 0 (Subnet ID) ou 255 (Broadcast)

Se outro valor: calcule o número mágico = 256 − máscara

Subnet ID do octeto: maior múltiplo do número mágico ≤ valor do IP nesse octeto

Broadcast do octeto: Subnet ID + número mágico − 1

🔢 O número mágico é o tamanho de cada sub-rede no octeto de interesse. Os subnet IDs são sempre múltiplos desse número: 0, N, 2N, 3N…
IP:192.168.5.77Máscara:255.255.255.224
Octeto interes.:4º (valor IP=77)
Nº mágico:256 − 224 = 32
Múltiplos:0, 32, 64, 96, 128…
≤77 mais próx.:64
Subnet ID:192.168.5.64
Broadcast:192.168.5.(64+32−1) = 192.168.5.95
Range:192.168.5.65 → 192.168.5.94
